# Изучение основ кредитного скоринга

## Цель проекта
Научиться предсказывать, вернёт ли человек кредит.  
Разобраться, как банки оценивают надёжность заёмщиков на основе их данных.

## План работы
1. Найти подходящий открытый датасет с историей кредитов (кто вернул, кто нет)
2. Провести разведочный анализ данных: понять, какие факторы влияют на возврат
3. Подготовить данные для моделирования
4. Построить модель логистической регрессии для предсказания дефолта
5. Оценить качество модели
6. Сделать выводы о важности различных признаков

## Данные
Планирую использовать один из открытых датасетов:
- [Give Me Some Credit](https://www.kaggle.com/datasets/crawford/give-me-some-credit)
- [German Credit Data](https://www.kaggle.com/datasets/uciml/german-credit) 

Датасет будет загружен и обработан в процессе работы над проектом.

## Навыки и опыт

### Базы данных и SQL
- PostgreSQL: создала базу данных из 12 таблиц с реальными связями между ними
- Практический опыт: проектирование структуры БД для медицинского учреждения (определение таблиц, первичных и внешних ключей)
- SQL-запросы: владею основными операциями выборки данных: разные типы JOIN (INNER, LEFT), агрегатные функции (COUNT, SUM, AVG), фильтрация (WHERE), сортировка (ORDER BY)
- Понимаю: связи между таблицами (один-ко-многим, многие-ко-многим), принципы построения запросов для получения нужной информации

### Программирование
